# Инструкция по тестированию системы регистрации и авторизации

## ✅ Что было исправлено
1. **Ошибка Prisma Client** - удалён неработающий адаптер libSQL, теперь используется стандартный PrismaClient для SQLite
2. **Верификация email** - добавлена полная система подтверждения email адреса

## 🆕 Добавленные функции

### Верификация Email
- При регистрации пользователь получает письмо с ссылкой для подтверждения email
- Токен верификации действителен 24 часа
- Страница подтверждения: `/verify-email?token=...`
- В dashboard показывается баннер для неподтвержденных пользователей
- Возможность повторной отправки письма с подтверждением

### Новые API Endpoints
1. **GET `/api/auth/verify-email?token=<token>`** - подтверждение email
2. **POST `/api/auth/resend-verification`** - повторная отправка письма
3. **GET `/api/user/me`** - получение полных данных пользователя
4. **GET `/api/test-db`** - тестовый endpoint для проверки работы БД

## 🧪 Как протестировать

### 1. Проверка работы сервера
Откройте в браузере:
- http://localhost:3001/ - главная страница
- http://localhost:3001/register - регистрация
- http://localhost:3001/login - вход

### 2. Проверка подключения к базе данных
```
http://localhost:3001/api/test-db
```
Должен вернуть: `{ "success": true, "message": "Prisma работает!", "userCount": 0 }`

### 3. Регистрация нового пользователя
1. Перейдите на http://localhost:3001/register
2. Заполните форму:
   - Имя: Андрей
   - Email: test@example.com
   - Пароль: test12345
   - ✓ Согласен с условиями
3. Нажмите "Зарегистрироваться"

**Ожидаемое поведение:**
- Пользователь создаётся в базе данных
- Генерируется JWT токен
- Попытка отправить письмо с верификацией (может не сработать без настройки SMTP)
- Редирект на onboarding

### 4. Проверка баннера верификации
1. После регистрации войдите на http://localhost:3001/dashboard
2. Должен показаться жёлтый баннер: "Пожалуйста, подтвердите ваш email адрес"
3. Есть кнопка "Отправить письмо повторно"

### 5. Проверка данных в базе (через Prisma Studio)
```powershell
cd pm-gym
npx prisma studio
```
Откроется веб-интерфейс для просмотра данных в БД

## 📧 Настройка Email (опционально)

Для работы отправки писем настройте в `.env`:
```env
EMAIL_SERVER_HOST="smtp.gmail.com"
EMAIL_SERVER_PORT="587"
EMAIL_SERVER_USER="ваш-email@gmail.com"
EMAIL_SERVER_PASSWORD="пароль-приложения-gmail"
EMAIL_FROM="PM Gym <noreply@pmgym.com>"
```

### Как получить пароль приложения Gmail:
1. Включите двухфакторную аутентификацию в аккаунте Google
2. Перейдите в: https://myaccount.google.com/apppasswords
3. Создайте новый пароль приложения
4. Используйте его в EMAIL_SERVER_PASSWORD

## 🐛 Если что-то не работает

### Ошибка "Port 3000 is already in use"
Сервер автоматически использует порт 3001

### Ошибка подключения к БД
1. Проверьте что файл `pm-gym/dev.db` существует
2. Выполните миграции:
```powershell
cd pm-gym
npx prisma migrate deploy
```

### Письма не отправляются
Это нормально без настройки SMTP. Токен верификации сохраняется в БД и можно:
1. Получить токен из базы через Prisma Studio
2. Вручную открыть: `http://localhost:3001/verify-email?token=<ваш-токен>`
