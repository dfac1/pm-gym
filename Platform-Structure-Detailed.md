## 1. Landing Page

Landing Page состоит из следующих секций, расположенных вертикально:

```
┌─────────────────────────────────────────────┐
│         HEADER (Navigation)                 │
├─────────────────────────────────────────────┤
│         HERO SECTION                        │
├─────────────────────────────────────────────┤
│         VALUE PROPOSITION                   │
├─────────────────────────────────────────────┤
│         HOW IT WORKS                        │
├─────────────────────────────────────────────┤
│         FEATURES PREVIEW                    │
├─────────────────────────────────────────────┤
│         LEARNING MODULES                    │
├─────────────────────────────────────────────┤
│         SOCIAL PROOF                        │
├─────────────────────────────────────────────┤
│         PRICING (FUTURE)                    │
├─────────────────────────────────────────────┤
│         FAQ                                 │
├─────────────────────────────────────────────┤
│         FINAL CTA                           │
├─────────────────────────────────────────────┤
│         FOOTER                              │
└─────────────────────────────────────────────┘
```

---

### 1.2 Header (Навигация)

**Позиция:** Fixed/Sticky top
**Фон:** Полупрозрачный с blur эффектом при скролле

**Структура:**
```
┌──────────────────────────────────────────────────────────┐
│  [LOGO] PM Gym     [О проекте] [Возможности] [Цены]     │
│                                    [Войти] [Начать →]    │
└──────────────────────────────────────────────────────────┘
```

**Элементы:**
- **Логотип + название:** PM Gym (слева)
- **Навигационное меню:**
  - О проекте (якорь на раздел)
  - Возможности (якорь на Features)
  - Цены (якорь на Pricing)
- **CTA кнопки:**
  - "Войти" (secondary кнопка) → переход на /login
  - "Начать бесплатно" (primary кнопка) → переход на /register

**Адаптивность:**
- Desktop: все элементы видны
- Tablet: меню сворачивается в hamburger
- Mobile: hamburger menu + только основная CTA кнопка

### 1.5 How It Works (Как это работает)

**Цель:** Показать процесс использования платформы

**Структура:**
```
┌──────────────────────────────────────────────────────────┐
│              Как это работает?                           │
│                                                          │
│  1 → 2 → 3 → 4 → 5                                       │
│                                                          │
│  [STEP 1]      [STEP 2]      [STEP 3]                    │
│  Выбери        Принимай      Получай                     │
│  сценарий      решения       результат                   │
│                                                          │
│  [Интерактивная визуализация процесса]                   │
└──────────────────────────────────────────────────────────┘
```

**5 шагов:**

1. **📝 Выбери сценарий**
   - Иконка: список кейсов
   - Описание: "Выбери реальную продуктовую ситуацию из библиотеки"

2. **🤔 Принимай решения**
   - Иконка: развилка
   - Описание: "Принимай решения в ветвящихся сценариях"

3. **📊 Смотри последствия**
   - Иконка: график
   - Описание: "Видишь как твои решения влияют на метрики"

4. **💡 Учись на разборе**
   - Иконка: лампочка
   - Описание: "Получай детальную обратную связь и альтернативы"

5. **🚀 Прокачивай навыки**
   - Иконка: рост
   - Описание: "Отслеживай развитие 8 ключевых компетенций"

**Визуализация:**
- Анимированный пример прохождения одного кейса
- Или интерактивная демка
- Прогресс-бар/стрелки между шагами

### 1.10 FAQ

**Цель:** Ответить на частые вопросы

**Формат:** Accordion (раскрывающиеся ответы)

**Вопросы:**

1. **Кому подходит PM Gym?**
   - Начинающим PM, которые хотят набраться опыта
   - Junior/Middle PM для прокачки конкретных навыков
   - Карьерным switchers в продакт-менеджмент
   - Студентам, готовящимся к интервью

2. **Нужен ли опыт для начала?**
   - Нет! Мы начинаем с базовых кейсов и постепенно усложняем.

3. **Сколько времени занимает обучение?**
   - Один кейс — 15-30 минут. Можно заниматься в своём темпе.

4. **Что если я ошибусь в кейсе?**
   - Это и есть суть обучения! Ты получишь детальную обратную связь и сможешь попробовать снова.

5. **Как PM Gym отличается от курсов?**
   - Это не пассивное обучение. Ты принимаешь реальные решения и видишь последствия.

6. **Получу ли я сертификат?**
   - Да, после прохождения определённого количества модулей (в Pro версии).

7. **Можно ли использовать для команды?**
   - Да, есть Team tier для корпоративного обучения (скоро).

8. **Бесплатная версия ограничена по времени?**
   - Нет, бесплатный доступ к базовым кейсам навсегда.

### 2.8 Design System для Auth страниц

**Layout:**
- Центрированная форма (max-width: 440px)
- Много whitespace
- Чистый, минималистичный дизайн

**Цвета:**
- Primary: яркий акцентный цвет для CTA
- Background: светлый/белый (light mode) или тёмный (dark mode)
- Text: высокий контраст для читаемости
- Error: красный (#E53E3E)
- Success: зелёный (#38A169)

**Типографика:**
- H1: 32px, bold
- Subtitle: 16px, regular, приглушённый цвет
- Labels: 14px, medium
- Input text: 16px (чтобы на iOS не зумился)
- Hints: 14px, приглушённый

**Buttons:**
- Primary: яркий цвет, white text, full width
- Secondary (Social): outline, иконка слева, full width
- Height: 48px (удобно для мобильных)
- Border radius: 8px
- Hover/Active states

**Inputs:**
- Height: 48px
- Border: 1px solid, neutral color
- Border radius: 8px
- Focus: outline с акцентным цветом
- Error state: красная рамка + сообщение снизу

**Spacing:**
- Между элементами: 16-24px
- Padding формы: 32px
- Margin снизу для inputs: 20px

**Responsive:**
- Desktop: центрированная форма, фоновая иллюстрация (optional)
- Mobile: full width контент, padding 16px по бокам

**Accessibility:**
- Все inputs с labels
- Tab navigation работает правильно
- Error messages объявляются screen readers
- Contrast ratio > 4.5:1

### 2.9 Auth Flow - Technical Requirements

**Security:**
- ✅ Password hashing (bcrypt)
- ✅ JWT tokens для сессий
- ✅ Refresh tokens
- ✅ CSRF protection
- ✅ Rate limiting (предотвращение bruteforce)
- ✅ Email verification (опционально для MVP)
- ✅ Social OAuth (Google, GitHub)

**Session Management:**
- Access token: 15 минут
- Refresh token: 7 дней
- "Remember me": 30 дней
- Auto logout при истечении

**Password Requirements:**
- Минимум 8 символов
- Рекомендация: заглавная + строчная + цифра + спецсимвол
- Проверка на leaked passwords (HaveIBeenPwned API)
- Password strength meter

**Email:**
- Отправка через транзакционный сервис (SendGrid/Postmark)
- Templates:
  - Welcome email
  - Email verification
  - Password reset
  - Login from new device (опционально)

**Analytics:**
- Трекинг каждого шага Auth flow
- A/B тесты на копирайт и layout
- Funnel analysis: Landing → Register → Onboarding → Dashboard
- Conversion rate оптимизация

**Error Handling:**
- Понятные сообщения об ошибках
- Не раскрывать, существует ли email в системе (для security)
- Логирование всех failed attempts
- Notifications для подозрительной активности

## 5. Технические требования

### 5.1 Performance

**Targets:**
- ⚡ First Contentful Paint < 1.5s
- ⚡ Time to Interactive < 2.5s
- ⚡ Lighthouse Score > 90
- ⚡ Core Web Vitals: все Green

**Optimization:**
- Code splitting по routes
- Lazy loading для компонентов
- Image optimization & lazy loading
- CDN для статики
- Service Worker для offline

### 5.2 Analytics & Tracking

**Events to track:**
- Page views
- User actions (clicks, form submissions)
- Learning progress (lesson start, complete)
- Scenario decisions & outcomes
- Time spent on pages
- Feature usage
- Errors & bugs

**Tools:**
- Google Analytics 4 (или альтернатива)
- Amplitude/Mixpanel для product analytics
- Custom events через API

### 5.3 Accessibility

**WCAG 2.1 Level AA compliance:**
- ✅ Keyboard navigation
- ✅ Screen reader support
- ✅ Proper ARIA labels
- ✅ Color contrast > 4.5:1
- ✅ Focus indicators
- ✅ Alt text для изображений
- ✅ Semantic HTML

### 5.4 SEO (для Landing)

- ✅ Semantic HTML (H1-H6 hierarchy)
- ✅ Meta tags (title, description)
- ✅ Open Graph tags
- ✅ Schema.org markup
- ✅ Sitemap.xml

- ✅ Fast loading

### 5.5 Security

- ✅ HTTPS только

- ✅ Rate limiting
- ✅ Input validation
